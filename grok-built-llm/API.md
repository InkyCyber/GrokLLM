# GrokLLM API Reference

All endpoints and documentation written autonomously by Grok Build CLI.

Base URL when running locally: `http://localhost:8000`

## Endpoints

### `GET /`

Serves the complete self-contained web interface (single HTML file with Tailwind via CDN).

### `GET /api/health`

Returns server and model status.

**Response 200**

```json
{
  "status": "ok",
  "model_loaded": true,
  "uptime_seconds": 124.3,
  "device": "cpu",
  "parameters": 809600,
  "checkpoint": "checkpoints/grokllm_tiny.pt",
  "attribution": "This server and model were created entirely by the Grok Build CLI in one autonomous session."
}
```

When no model is present:

```json
{
  "status": "no_model",
  "model_loaded": false,
  ...
}
```

### `POST /api/generate`

The primary generation endpoint.

**Request body**

```json
{
  "prompt": "The lady doth protest too much",
  "max_new_tokens": 180,
  "temperature": 0.75,
  "top_k": 40,
  "top_p": 0.9,
  "seed": null
}
```

**Response 200**

```json
{
  "text": "The lady doth protest too much, methinks.\n\nHAMLET:\n...",
  "prompt": "The lady doth protest too much",
  "generated_continuation": "\n\nHAMLET:\n...",
  "tokens_generated": 180,
  "elapsed_seconds": 1.24,
  "tokens_per_second": 145.2,
  "parameters": {
    "max_new_tokens": 180,
    "temperature": 0.75,
    "top_k": 40,
    "top_p": 0.9,
    "seed": null
  },
  "model_info": {
    "parameters": 809600,
    "block_size": 128,
    "n_layer": 4
  },
  "attribution": "This response was generated by a model trained and served by code written entirely by Grok Build CLI."
}
```

**Error cases**
- `503` — No model loaded (train first)
- `500` — Internal generation error

## OpenAPI / Swagger

Visit `/docs` when the server is running for an automatically generated Swagger UI.

## Example: curl one-liner

```bash
curl -s http://localhost:8000/api/generate \
  -X POST \
  -H "Content-Type: application/json" \
  -d '{"prompt":"All the world's a stage","max_new_tokens":90,"temperature":0.7}' \
  | jq -r '.text'
```

## Notes

- All generation is **synchronous**. For this model size there is no noticeable benefit to streaming.
- The model is loaded once at process start. Restart the server after training a new checkpoint.
- No authentication or rate limiting is implemented (local demo only).
